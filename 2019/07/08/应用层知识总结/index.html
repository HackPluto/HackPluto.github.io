应用层知识总结

重点知识点：
DHCP ★★★★
DNS ★★★★★
HTTP ★★★★★
WWW ★★★★★

一.引导协议BOOTP：

1.BOOTP是RATRP的替代协议，解决了RARP只能返回IP地址和不能被路由器转发的不足（迫使每一个局域网都得建立一个RARP服务器）
2.基于UDP，要求计算校验和，客户端68端口，服务器67端口，通常和TFTP协同工作。
3.无盘工作站启动时，用有限广播在本网广播BOOTP请求报文。
4.客户端广播请求，服务器可以广播回应也可以在本机内设置一个ARP缓存然后定向单播应答。
5.使用超时重传，重传使用指数退避策略。
6.请求和响应报文不能分片（300字节）。
7.如何使用路由器转发BOOTP：
通常用在本网没有BOOTP服务器的情况下
路由器此时称作中继代理
客户机先将数据包发给路由器，路由器收到包后将自己的IP填入路由器IP项，跳数加1，单薄给BOOTP服务器
服务器响应和上述过程相反，当跳数为3时丢弃

二.动态主机配置协议DHCP：

1.动态配置必要性
BOOTP需要人工进行协议参数的配置
BOOTP适用于相对静态的网络
IP地址不足，所有人同时上网的可能性不大，适合动态分配IP地址

2.DHCP组成三要素：
DHCP服务器
DHCP Relay（路由器，交换机）
DHCP客户端

3.DHCP三种地址分配策略：
1.手动配置 为某一个特定主机分配一个IP
2.自动配置 为某一个新上线的主机分配一个永久IP地址
3.全动态配置 为上网主机临时分配一个IP地址
存放所有可分配IP的地方叫地址池

4.客户状态转换：
DHCP DISCOVER 是客户端使用DHCP时发送的第一个报文
DHCP OFFER 是服务器向客户端DISCOVER的响应
DHCP REQUEST 是在服务器选择的过程中客户端对OFFER的响应，也可以作为客户端延长IP地址租期的请求
DHCP ACK 服务器对REQUEST的响应，包含了客户端IP地址的信息
DHCP NACK 服务器对REQUEST的拒绝响应，客户端一般会重新开始DHCP请求
DHCP RELEASE 客户端主动释放分配给他的IP地址
DHCP DECLINE 客户端发现分配给他的IP地址不能用，发送此报文
DHCP INFORM 客户端从服务器获得DNS，网关等其他网络信息

5.IP使用租期：
每一个客户端动态分配到的IP都有一个租期T，当一个客户端获取到IP时会设置3个计时器，分别是0.5T，0.875T，T。当到达0.5T时，客户端发送 DHCP REQUEST请求增加租期，如果服务器回复ACK就继续使用，并且更新计时器；如果服务器发送NACK就立即停止使用；如果服务器不回复就达到0.875时再发送请求。

6.DHCP欺骗攻击：
攻击者先发送大量的IP请求包耗光DHCP服务器的IP，然后当有计算机请求IP时，攻击者伪造响应，发送给被攻击者错误的地址等，通过DHCP欺骗的方式进行中间人攻击。

三.域名系统DNS：

1.DNS出现之前计算机使用主机上的Hosts文件进行域名的解析
2.层次型命名机制 引入结构，分层命名，按层分布式管理

3.常见顶级域
.com– 用于商业机构。它是最常见的顶级域名。
.net– 最初是用于网络组织，
.org– 是为各种组织包括非盈利组织而定的。
.edu–用于教育机构。
.mil–用于军事部门
.gov–用于政府部门
.biz—-用来替代.com的顶级域，适用于商业公司； (注：biz是business的习惯缩用)；
.info—用来替代.com的顶级域，适用于提供信息服务的企业；
.name—专用于个人的顶级域； .pro—-专用于医生、律师、会计师等专业人员的顶级域; (注：pro是professional的习惯缩用)； .coop—专用于商业合作社的顶级域；（注：coop 是cooperation的习惯缩用）；
.aero—专用于航空运输业的顶级域名；
.museum—专用于博物馆的顶级域。

4.域名解析原理：
域名服务器：提供名字-地址转换的服务器软件
域名解析器：提供名字-地址转换的客户端软件

5.域名解析方式：
反复解析 在自己不能解析的情况下告诉客户端谁可以来解析
递归解析 在自己不能解析的情况下代替客户端去解析（根域名服务器不进行递归解析）

6.反向解析：
由IP查询域名 构造一个特别的域in-addr.arpa，指针查询表达式：aaa.bbb.ccc.ddd.in-addr.arpa

7.DNS解析过程：
用户在浏览器输入URL
浏览器向一个DNS服务器发起一个递归查询直到返回一个A类记录
DNS服务器向浏览器返回一个A类记录aaa.bbb.ccc.ddd的IP地址
浏览器向aaa.bbb.ccc.ddd发起请求

8.DNS使用TCP还是UDP：
DNS一般使用UDP的53端口但是当响应报文长度大于512字节的时候，只使用UDP返回前512字节，后面的使用TCP传输。
当一次解析的域名很多时会使用TCP直接发起查询。
主，辅域名服务器进行区域传送

四.超文本传输协议HTTP
1.www要解决的问题

两个问题：
能否不使用物理地址而使用内容进行查询
能否用统一的方式对各种资源进行标注
两个概念：
超文本：使用超链接的方式将不同空间的信息链接在一起的网状文本
统一资源定位符URL：对所有资源的统一标示
两项实现技术：
超文本传输协议 HTTP
超文本标记语言 HTML

2.B/S架构

浏览器/服务器架构
特点：
用户通过浏览器去访问各种信息，这些信息都存放在web服务器，服务器在后端又连接着很多数据库，大量的数据实际上存在于大量的数据库中。
优点：
1.分布式，可以随时查询，浏览
2.业务拓展简单
3.维护简单方便
4.开发简单，共享性强
缺点：
1.个性化功能差
2.页面动态刷新，响应效率低
3.操作以鼠标为主，不能实现快速操作
4.不能实现分页显示，数据库压力大（分页显示就是在SQL查询语句中限制页数以及字段数）

3.HTTP定义

HTTP全称超文本传输协议，建立在可靠的网络层协议基础上，提供了客户端和服务器之间传输信息的一种机制，规定了客户端和服务器交互的各种信息。
HTTP功能：
1.文件下载（主要功能）
2.WEB交互

4.HTTP特点

1.C/S模型
2.简单快速
3.内容协商
4.可扩充性
5.无持久性连接
6.无状态

5.常见状态码

100 请重发请求
200 一切正常
303 请跟另一资源交互 重定向
304 客户端已有该数据
400 请求格式有问题
401 验证失败
404 请求错误
409 服务器拒绝资源更新请求
412 请求未被处理
500 服务器出错
