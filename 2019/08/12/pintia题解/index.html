7-51

题目的算法不是很难，就是定义两个指针分别指向1，2链表，然后迭代比较直到一个遇到了NULL为止。主要是自己很久都没有用链表，把这个数据结构有点忘了，说一个自己做的时候遇到的坑。定一个一个头节点指针后，因为定义的是指针类型，所以要使用被调函数执行插入操作的话，传入的应该是一个二级指针，如果不使用被调函数就一级指针就好了，这也很好理解，毕竟C按值传递的形参的值改变了并不会影响实参，寄存器只能直接寻址。这里我也考虑过使用C++的引用但是也遇到了同样的问题。

7-18

题目很简单，直接使用C++中的STL库，定义两个队列，存入数据，按照2:1的比例pop即可。

水题

7-33

很明显一个图论的题，因为题目也没有说是稀疏图还是稠密图所以为了追求计算的效率，我使用邻接矩阵存储。
题目的解法在题干中也说了，就是DFS，就使用普通的图的遍历即可，题目还说要输出路径，本来我想的是将路径存储下来，遍历结束的时候输出，后来发现有更简单的做法，在DFS的时候就输出当前节点，返回的时候也输出当前节点，就做到了正向遍历再逆序输出的效果。DFS的终点就是没有点可以走的时候或者周围的点都是走过的点的时候。

6-10

题目意思很明确，就是一个二分查找，需要注意的是需要从1开始而不是从0开始，然后就是非常基本的二分查找。
有的同学把所有的代码都复制下来，然后运行，这肯定是会报错的，因为题目中List ReadInput();是错的，这一个函数没有写出返回值，所以链接器会报错。

7-37

一个排序题，题目算法不难，就是比较麻烦，麻烦在需要比的东西很多。首先说说数据结构，肯定是要用结构体存的，这样排序的时候才可以一个结构体一个移动。
再说说排序的问题，对于学号的排序，题目中是要用字符串存储，字符串比较的话我不知道直接用cmp函数能不能这种数字的字符，我试用了另一种方法，sscanf函数，这个函数功能十分强大。可以将数据重定向到别的变量里，所以可以将一个字符串重定向到一个整形变量里。
对于姓名我使用string存储，因为string类的方法更多，可以直接比大小。
sort排序可以自定义排序策略，例如代码中的compare_id函数，这种函数的格式就是固定的。

7-20

这个题是对栈的应用。因为栈有FILO的特性，所以可以做到表达式的转换。需要注意的是栈当中只保存操作符，因为真正决定是哪种表示式的是操作符，数字的顺序是一定不会变的。所以如果遇到操作数，就直接输出。如果遇到操作符，则将其压入到栈中。对于括号需要特殊处理，因为括号是可以强行改变运算符优先级的，右括号代表着一个优先级的结束所以，遇到左括号时也将其压入栈中，遇到一个右括号，则依次将栈顶元素弹出，并输出，直到遇到左括号为止。
对于别的操作符，如（”+”， “*”，”(“）等，从栈中弹出元素直到遇到发现更低优先级的元素(或者栈为空)为止。弹出完这些元素后，才将遇到的操作符压入到栈中。有一点需要注意，只有在遇到” ) “的情况下才弹出” ( “，其他情况都不会弹出” ( “。
弹出运算符时，遵循优先级高的优先弹出。在左括号压栈之前，其优先级最高，压栈之后，其优先级最低。如果我们读到了输入的末尾，则将栈中所有元素依次弹出。

7-2

这个题最开始读题的时候一直没有读懂，后来才看懂题目的输入和输出,就是将一串数字，两两一组，前面的是系数后面的事指数，比如3 5就是3*x^5,然后就是两组数的乘法或者加法运算。
这种题目在数据结构的选择上肯定是数组的，数组不仅访问快而且取下标很方便，可以将对应的下标内的值变成他的系数。
在计算的部分就是，使用一个二重循环，迭代每一个数，乘或者加他们的指数和系数。这个代码我觉得唯一的缺点就是时间复杂度太大，不知道有没有时间复杂度小的算法。

7-19
这个题我做的有点投机取巧吧，题目叫链式表，但是我怕链表的时间复杂度太大，就使用了数组，并且题目中也没有明确指出需要使用数组。对于数组的大小，题目中说输入一个正整数k，说明数组的大小是小于int类型最大值的，所以就使用了10^6。然后在无效输入部分就是，k<0或者k>输入的个数。

7-24

这个题还是很有意思的，考查了二叉排序树，因为题目中需要将树的名字按照ASCII的顺序输出，所以不能用普通的数据结构来存储，必须使用二叉排序树，但是二叉排序树又容易形成链表，所以这里最好使用平衡的二叉排序树，也就是红黑树。C++里已经有了基于红黑树的数据结构，就是map，所以这个题我最后选用了c++里的map。题目本身没有什么算法，就是map一个遍历的过程，但是输入数据的时候需要注意，不能使用普通的scanf或者cin因为树名存在空格，这里需要使用getline，这个函数以换行符作为结尾。

7-42

这是一道哈希映射的题目，唯一的难点可能就是线性探测法的定义，就是如果计算出的哈希值已经被使用，就向后查找，如果后面都没有就从开头再向后查找。题目中的模数是p，所以可能的哈希值就是[0,p-1]，因为N<1000，所以P最大是1009，所以哈希的数组设置成1010的容量即可。因为题目中可能出现相同元素的情况，对于相同元素只需要计算哈希一次，所以需要记录一个数有没有被使用，这里我使用一个book数组，记录使用的情况。

7-38

一个简单的排序题，题目并不需要对所有数排序只需要找出前k名，即可，而k最大也就是10。所以最多排序10次即可，这里我选择直接排序10次，然后输出前k个。但是怎么排序出前10个，麻烦一点可以定义10个变量然后一遍一遍的遍历，这样时间复杂度倒不大但是代码实在是太丑了很冗余。我的方法是这样的，因为存在相同元素的情况，所以把num数组再开一维来标记这个数有没有使用过，需要一个变量last来记录上一个被标记的数，当发生数的交换时就将num[last][1]=0;同时num[j][1]=1;同时还有变量M，来记录上一次循环的最大值，找出第三大的数并不需要让他和第一大第二大都比较下，只需要比第二大小于等于即可，因为我们在做第二大数的选择时同样是这样的策略。
